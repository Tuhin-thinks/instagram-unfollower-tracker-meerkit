# Security Policy

## Supported Versions

Meerkit is currently pre-1.0. Security fixes are applied on the latest branch only.

| Version | Supported |
| ------- | --------- |
| 0.1.x   | Yes       |
| < 0.1.0 | No        |

## Reporting a Vulnerability

Please do not open public issues for security vulnerabilities.

Use one of the following channels:

1. Preferred: Open a private security advisory in this repository:
   https://github.com/Tuhin-thinks/meerkit/security/advisories/new
2. If private advisory is unavailable, open an issue and clearly mark it as SECURITY so maintainers can triage quickly.

## What to Include

To help with fast triage, include:

- Affected component (backend, frontend, auth flow, dependencies, etc.)
- Steps to reproduce
- Impact and severity estimate
- Proof of concept (if safe)
- Suggested mitigation (optional)

## Response Targets

- Initial acknowledgement: within 3 business days
- Triage decision: within 7 business days
- Fix timeline: depends on severity and complexity

## Disclosure

Please allow maintainers time to investigate and patch before public disclosure.

After a fix is released, responsible disclosure with technical details is welcome.
